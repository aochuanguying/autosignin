## 🎯 系统概述

一汽奥迪自动化程序是一套基于 AutoJS6 的 Android 自动化脚本系统，实现以下功能：

- ✅ **一汽奥迪自动签到** - 每日自动签到赚取积分
- ✅ **一汽奥迪自动发帖** - 远程生成内容并自动发布到社区
- ✅ **RQrun 签到** - RQrun App 自动签到
- ✅ **信鸿 3.0 打卡** - 上班/下班自动打卡（含随机延迟）
- ✅ **失败告警** - 任务失败时通过 Bark 推送 + 电话拨号告警
- ✅ **成功打点** - 业务成功后写入标记文件，供告警脚本判断

## 🔧 核心脚本说明

### 1. audi_signin.js - 一汽奥迪签到

**功能**: 自动打开一汽奥迪 App，进入"我的"页面，点击"每日签到"

**核心流程**:
1. 检查成功标记文件，如存在则跳过
2. 解锁屏幕，亮屏并保持唤醒 5 分钟
3. 强制停止一汽奥迪 App（清理残留状态）
4. 启动 App 并等待首页就绪（底部导航出现）
5. 点击"我的"Tab，进入我的页面
6. 校验登录账号是否为"王大锤"
7. 查找"每日签到"入口
8. 点击签到，等待 10 秒
9. 写入成功打点文件
10. 清理：强杀 App、回桌面、熄屏

**配置项**:
```javascript
var CONFIG = {
  appPackage: "com.timanetworks.android.frame.audisuper.release",
  appName: "一汽奥迪",
  mineTabId: "navigation_main_fifth_tab",
  minePageReadyId: "personal_name",
  signinText: "每日签到",
  startupAdWaitMs: 12000,        // 广告页等待超时
  waitAfterSigninMs: 10000,      // 签到后等待
  useRoot: true,                 // 使用 Root 权限
  screenOffAfterRun: true,       // 运行后熄屏
  keepScreenOnMs: 300000         // 保持亮屏 5 分钟
};
```

**成功打点文��**: `/sdcard/脚本/audi_signin_success.log`

### 2. audi_post.js - 一汽奥迪发帖

**功能**: 从远程服务器获取发帖内容（标题、正文、图片），自动发布到一汽奥迪社区

**核心流程**:
1. 检查成功标记文件，如存在则跳过
2. 设备准备（亮屏、保持唤醒 20 分钟）
3. 启动一汽奥迪 App，进入"我的"→"赚取奖励"→"去完成"
4. **远程获取发帖内容**:
   - 调用异步批量生成接口 `/api/posts/batch`
   - 轮询任务状态（静默期 90s + 分段轮询）
   - 超时后降级到同步生成接口 `/api/posts/generate`
5. **下载图片**:
   - 下载远程图片到本地 `/sdcard/Pictures/AUDI`
   - 使用 Root 权限修复媒体扫描问题
6. **填写内容**:
   - 清空编辑器
   - 填入标题、正文、话题
   - 执行录制动作（`audi_post.auto`）上传图片
7. **等待上传完成**:
   - 轮询检测图片上传进度
   - 支持稳定期兜底（60 秒无变化视为成功）
8. **发布帖子**:
   - 点击发布按钮
   - 处理二次确认（如有）
   - 检测"审核中"状态
9. **回调确认**:
   - 成功：调用 `/api/posts/confirm` 确认发布成功
   - 失败：回调告知服务端发布失败
10. 写入成功打点文件
11. 清理：强杀 App、回桌面、熄屏

**配置项**:
```javascript
var CONFIG = {
  appPackage: "com.timanetworks.android.frame.audisuper.release",
  appName: "一汽奥迪",
  imageDir: "/sdcard/Pictures/AUDI",
  recordedAutoFile: "/sdcard/脚本/audi_post.auto",
  remoteBaseUrl: "https://yqad.hxfssc.com:8088/api",
  remoteToken: "api_token_xxx",
  remoteGeneratePath: "/posts/generate",
  remoteBatchPath: "/posts/batch",
  remoteConfirmPath: "/posts/confirm",
  remoteUseAsync: true,          // 使用异步批量生成
  remoteTimeoutSec: 600,         // HTTP 超时 10 分钟
  remotePollTimeoutMs: 600000,   // 轮询超时 10 分钟
  remotePollSchedule: [          // 分段轮询策略
    { endMs: 90000, intervalMs: 0, allowRequest: false },    // 静默期 90s
    { endMs: 240000, intervalMs: 20000, allowRequest: true }, // 2-4 分钟每 20s 轮询
    { endMs: 600000, intervalMs: 10000, allowRequest: true }  // 4-10 分钟每 10s 轮询
  ],
  remoteEnableSyncFallback: true,     // 异步失败降级同步
  remoteFallbackOnTimeout: true,      // 超时降级
  remoteFallbackOnTaskFailed: true,   // 任务失败降级
  keepScreenOnMs: 1200000        // 保持亮屏 20 分钟
};
```

**成功打点文件**: `/sdcard/脚本/audi_post_success.log`

### 3. auto_call.js - 告警拨号

**功能**: 检查各任务成功打点文件，如存在失败则发送 Bark 推送并拨打电话告警

**核心流程**:
1. 检查串行调度锁文件，如存在则跳过
2. 检查成功打点文件:
   - `audi_signin_success.log`
   - `rqrun_signin_success.log`
   - `audi_post_success.log`
3. **全部成功**: 不发送通知，直接退出
4. **存在失败**:
   - 构建告警消息（列出失败任务）
   - 发送 Bark 推送通知到 iOS 设备
   - 调用 Telecom API 拨打电话告警
5. 清理：熄屏

**配置项**:
```javascript
var CONFIG = {
  barkUrl: "https://api.day.app/Asbu4fr2HjGAjKbHANNbLS",
  iconUrl: "https://sf16-passport-sg.ibytedtos.com/img/user-avatar-alisg/...",
  phoneNumber: "18953272532",     // 告警电话号码
  apiBaseUrl: "http://localhost:5000",  // Telecom API 地址
  screenOffAfterRun: true
};
```

**Bark 推送格式**:
- 标题：`签到任务异常`
- 内容：`【奥迪签到】失败、【RQrun 签到】失败、【奥迪发帖】失败`

**电话拨号**: 调用 Android Telecom API `/api/v1/call` 接口

## 📡 发帖回调机制

### 回调流程

```
1. 生成内容          2. 发布帖子          3. 回调确认
   ┌─────┐            ┌─────┐            ┌─────┐
   │ API │───►        │ Auto│───►        │ API │
   │生成 │            │ JS  │            │确认 │
   └─────┘            └─────┘            └─────┘
      │                  │                  │
      ▼                  ▼                  ▼
   taskId             审核中            success=true
```

### 回调接口

**成功回调**:
```bash
POST https://yqad.hxfssc.com:8088/api/posts/confirm
Authorization: Bearer api_token_xxx
Content-Type: application/json

{
  "taskId": "post_1718362800000_abc123",
  "postId": "community_post_456",  // 可选
  "success": true
}
```

**失败回调**:
```bash
POST https://yqad.hxfssc.com:8088/api/posts/confirm
Authorization: Bearer api_token_xxx
Content-Type: application/json

{
  "taskId": "post_1718362800000_abc123",
  "success": false
}
```

### 回调时机

| 场景 | 回调时机 | success 值 |
|------|---------|-----------|
| 发布成功 | 检测到"审核中"状态 | `true` |
| 发布失败 | 未检测到成功状态 | `false` |
| 图片上传超时 | 等待上传超时 | `false` |
| 录制回放失败 | 未回到编辑页 | `false` |
| 脚本异常 | catch 块中回调 | `false` |

### 回调响应

**成功响应**:
```json
{
  "success": true,
  "topicId": "topic_abc123",
  "remainingUses": 2
}
```

**字段说明**:
- `topicId`: 使用的主题 ID
- `remainingUses`: 主题剩余使用次数

## 📦 部署指南

### 步骤 1：准备 AutoJS6 环境

1. 在 Android 设备安装 AutoJS6
2. 授予无障碍权限
3. 授予 Root 权限（设置 → 高级设置 → Root 授权）

### 步骤 2：复制脚本到设备

```bash
# 推送脚本到手机
adb push /Users/mac/Documents/workspace/krio/autosignin/scripts/ /sdcard/脚本/

# 赋予执行权限
adb shell "chmod +x /sdcard/脚本/*.js"
adb shell "chmod +x /sdcard/脚本/*.py"
```

### 步骤 3：配置参数

编辑脚本中的 `CONFIG` 对象，修改以下配置：

**audi_post.js**:
```javascript
var CONFIG = {
  remoteBaseUrl: "https://yqad.hxfssc.com:8088/api",
  remoteToken: "api_token_1640a8b188784e52e08e11eb8dcab3a9fcea5a8d6b03e1235d6705938eed853a",
  imageDir: "/sdcard/Pictures/AUDI",
  recordedAutoFile: "/sdcard/脚本/audi_post.auto"
};
```

**auto_call.js**:
```javascript
var CONFIG = {
  barkUrl: "https://api.day.app/你的 Bark 密钥",
  phoneNumber: "你的告警电话号码",
  apiBaseUrl: "http://安卓设备 IP:5000"
};
```

### 步骤 4：录制图片上传动作

**audi_post.auto** 需要使用 AutoJS6 的录制功能：

1. 打开 AutoJS6 → 录制动作
2. 手动执行一次图片选择、上传流程
3. 保存为 `/sdcard/脚本/audi_post.auto`

### 步骤 5：配置定时任务

在 AutoJS6 中配置定时任务：

```javascript
// 示例：每天 9:00 执行签到

// 每天 18:00 执行发帖
auto.setTimer("0 0 18 * * *", "audi_post.js");

// 每天 00:05 清理成功标记

// 每小时检查一次任务状态
auto.setTimer("0 * * * * *", "auto_call.js");
```

### 步骤 6：测试运行

```bash

adb shell "su -c 'cd /sdcard/脚本 && /data/data/org.autojs.autojs6/autojs6 audi_signin.js'"
```

## 🔍 故障排查

### 1. 脚本无法启动

**问题**: AutoJS6 提示"无障碍服务未启用"

**解决**:
1. 打开 Android 设置 → 无障碍
2. 启用 AutoJS6 Accessibility Service
3. 重启 AutoJS6 App

---

### 2. 签到失败：首页未就绪

**问题**: 日志显示"首页未就绪，可能仍停留在广告页"

**解决**:
1. 增加 `startupAdWaitMs` 配置（如改为 15000ms）
2. 检查一汽奥迪 App 是否最新版本
3. 手动打开 App 查看是否有弹窗广告

---

### 3. 发帖失败：图片上传超时

**问题**: 日志显示"图片上传等待超时"

**解决**:
1. 检查 `/sdcard/Pictures/AUDI` 目录是否存在
2. 检查媒体扫描是否完成：
   ```bash
   adb shell "su -c 'pm clear com.android.providers.media'"
   ```
3. 重新录制 `audi_post.auto` 文件
4. 增加 `uploadWaitTimeoutMs` 配置

---

### 4. 告警未触发

**问题**: 任务失败但未收到 Bark 推送

**解决**:
1. 检查 Bark URL 是否正确
2. 测试 Bark 推送：
   ```bash
   curl "https://api.day.app/你的密钥/测试/这是一条测试消息"
   ```
3. 检查 `auto_call.js` 是否被定时任务触发
4. 查看 AutoJS6 日志：
   ```bash
   adb logcat | grep auto_call
   ```

### 5. Telecom API 拨号失败

**问题**: 日志显示"电话拨出失败"

**解决**:

   ```bash
   adb shell "su -c 'ps -A | grep server.py'"
   ```

### 6. 成功打点文件未写入

**问题**: 脚本执行成功但未生成 `_success.log` 文件

**解决**:
1. 检查 `/sdcard/脚本/` 目录权限：
   ```bash
   adb shell "ls -la /sdcard/脚本/"
   ```
2. 手动创建目录：
   ```bash
   adb shell "mkdir -p /sdcard/脚本"
   ```
3. 查看脚本日志，确认是否执行到写入步骤

### 7. 媒体扫描失败（Android 15）

**问题**: 图片下载后无法在相册中显示

**解决**:
使用 Root 方案修复媒体扫描：

```bash
# 1. 修改文件权限
adb shell "su -c 'chmod 644 /sdcard/Pictures/AUDI/*.jpg'"

# 2. 修改文件所有者
adb shell "su -c 'chown media_rw:media_rw /sdcard/Pictures/AUDI/*.jpg'"

# 3. 清除媒体存储数据
adb shell "su -c 'pm clear com.android.providers.media'"

# 4. 触发媒体扫描
adb shell "su -c 'content call --uri content://media --method scan_volume --arg external_primary'"
```
