# 倍控 G31 N7305 旁路由完整部署方案

## 概述

- **设备**: 倍控 G31，Intel Celeron 7305 (5核6线程, 1.00GHz)，4×2.5G i226-V 网口 + 1×RJ45 COM 串口，1T SSD，4G DDR4
- **目标系统**: Debian 13 (Trixie) minimal
- **角色**: 专职旁路由 — 透明代理 + DNS 分流 + 去广告
- **新 IP**: 192.168.50.2（接替原 iStoreOS 旁路由）
- **网段**: 192.168.50.0/24，主路由 192.168.50.1（华硕 BE86U）

## 网络拓扑

```
光猫(桥接) → BE86U WAN(2.5G) 
                │
                ▼
        BE86U LAN(2.5G) → 兮克交换机(2.5G)
                                │
                ├── 倍控 G31 eth0 (192.168.50.2) ← 旁路由
                ├── 群晖 218+ (2.5G口)           ← NAS
                ├── 联想 X5-14 (1G口)            ← 服务器
                └── 其他有线设备
```

## 第一部分：通过 VirtualBox 安装 Debian 13 并写入 SSD

利用联想 X5-14（Win11）的 VirtualBox 虚拟机安装配置好 Debian，最后将磁盘镜像写入倍控的 SSD，无需键盘和显示器连接倍控。

> **方案说明**：VirtualBox 的 raw disk 直通在 Windows 11 + USB 硬盘盒环境下有严重的写入权限兼容性问题
> （`BLKCACHE_IOERR / VERR_ACCESS_DENIED`），即使磁盘已脱机、只读已清除仍然报错。
> 本方案改为：先在普通虚拟磁盘中安装配置 Debian → 导出 raw 镜像 → dd 写入 SSD。

### 1.1 在联想上下载所需软件

**VirtualBox 7.2**：https://www.virtualbox.org/wiki/Downloads → 下载 "Windows hosts" 版本（当前 7.2.12），安装。

**Debian 13 ISO**：https://mirrors.tuna.tsinghua.edu.cn/debian-cd/current/amd64/iso-cd/
下载最新的 `debian-13.x.x-amd64-netinst.iso`（约 800MB，选第一个 netinst 文件，不要 edu 或 mac 版本）。

**dd 写入工具**（后续步骤需要，提前下载）：
- [balenaEtcher](https://etcher.balena.io/)（推荐，选文件 → 选磁盘 → 烧录，三步搞定）
- 或使用 WSL 中的 dd 命令

### 1.2 创建虚拟机

打开 VirtualBox（无需管理员身份），点击「新建」(New)：

> VirtualBox 7.2 建议切换到 **Expert 模式**（向导底部有切换按钮）。

**Name and Operating System 页面：**
- 名称 (Name)：`gateway`
- ISO Image：选择下载好的 Debian 13 netinst ISO
- Type：Linux / Subtype：Debian / Version：Debian (64-bit)
- **取消勾选** "Skip Unattended Installation"（如已勾选）— 我们要手动安装

**Hardware 页面：**
- Base Memory：2048 MB
- Processors：2
- **勾选 "Enable EFI"**（倍控使用 UEFI 启动，必须勾选）

**Virtual Hard Disk 页面：**
- 选择 **"Create a Virtual Hard Disk Now"**
- 磁盘大小：**50 GB**（够装系统和所有配置，后续写入 SSD 后再扩展）
- 格式保持默认 VDI，动态分配

创建完成后，进入虚拟机「设置」(Settings)：
- **网络** (Network) → 连接方式：**桥接网卡** (Bridged Adapter)

### 1.3 安装 Debian 13

启动虚拟机，选择 "Graphical Install"：

| 步骤 | 选择 |
|------|------|
| Language | English |
| Location | China |
| Keyboard | American English |
| Hostname | `gateway` |
| Domain | 留空 |
| Root password | 设置强密码（**设置了 root 密码后，安装器会跳过创建普通用户**） |
| User | 如果安装器仍要求创建普通用户，可以创建一个然后之后禁用 |
| Partitioning | **Guided - use entire disk** |

> **Debian 13 (Trixie) 安装注意事项**：
> - 设置了 root 密码 → 不会创建普通用户。不设置 root 密码 → 锁定 root 并给普通用户 sudo。
>   本方案选择设置 root 密码、直接用 root 操作。
> - 安装器默认使用 deb822 格式源文件（`/etc/apt/sources.list.d/debian.sources`）。

### 1.4 配置兜底网络（关键！）

安装完成后虚拟机会重启进入 Debian，用 root 登录，执行：

```bash
# 禁用 ifupdown 的固定网口配置
cat > /etc/network/interfaces << 'EOF'
auto lo
iface lo inet loopback
EOF

# 用 systemd-networkd 匹配所有以太网口自动 DHCP
mkdir -p /etc/systemd/network
cat > /etc/systemd/network/20-wired.network << 'EOF'
[Match]
Type=ether

[Network]
DHCP=yes
EOF

# 启用 systemd-networkd
systemctl enable systemd-networkd

# 确认 SSH 开机启动 + 允许 root 登录
sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config
systemctl enable ssh
```

> **为什么需要兜底网络**：虚拟机里的网口名（如 enp0s3）和倍控真实网口名（如 enp1s0）不同。
> 用 systemd-networkd 匹配 `Type=ether` 可以确保不管网口叫什么名字都能自动 DHCP 获取 IP。
> SSD 插到倍控后，只要网线连着就能通过 SSH 连接。

接下来在虚拟机中继续完成第二部分~第九部分的所有配置，全部配好后再导出写入 SSD。

### 1.5 导出磁盘镜像并写入 SSD

> ⚠️ 在虚拟机中完成**所有配置**（第二~九部分）后再执行此步骤！

#### 步骤一：关闭虚拟机

```bash
poweroff
```

#### 步骤二：导出为 RAW 镜像

在联想的 PowerShell 中执行：

```powershell
cd "C:\Program Files\Oracle\VirtualBox"

# 找到你的 VDI 文件路径（通常在 VirtualBox VMs 目录下）
.\VBoxManage clonehd "C:\Users\你的用户名\VirtualBox VMs\gateway\gateway.vdi" "C:\Users\你的用户名\Desktop\gateway.img" --format RAW
```

> 这会生成一个约 50GB 的 raw 镜像文件（实际大小取决于你在虚拟机中写入了多少数据）。
> 确保桌面或目标路径所在磁盘有足够空间。

#### 步骤三：将镜像写入 SSD

**方法 A — 使用 balenaEtcher（推荐）：**

1. SSD 硬盘盒通过 USB 连接联想电脑
2. 打开 balenaEtcher
3. 点「从文件烧录」→ 选择 `gateway.img`
4. 点「选择目标磁盘」→ 选择 1TB SSD
5. 点「现在烧录！」，等待写入完成

> 如果磁盘之前有残留分区（如 EFI 分区），先用 diskpart 清除：
> `diskpart` → `select disk 1` → `clean` → `exit`

**方法 B — 使用 WSL 的 dd：**

```bash
# 在 WSL (Ubuntu) 中执行
# 先确认 SSD 的设备路径
wmic diskdrive list brief
# 找到 SSD 对应的 PhysicalDrive 号

# 写入（替换 PhysicalDrive1 为实际编号）
sudo dd if=/mnt/c/Users/你的用户名/Desktop/gateway.img of=/dev/sdb bs=4M status=progress
```

> ⚠️ **确认目标磁盘正确！dd 写错目标会覆盖系统盘，不可恢复。**

#### 步骤四：写入完成后扩展分区

SSD 插入倍控、SSH 连接后，50GB 的分区需要扩展到整块 1TB SSD：

```bash
apt install -y parted e2fsprogs

# 查看当前分区布局
lsblk

# 扩展第2个分区（ext4 根分区）到磁盘末尾
parted /dev/sda resizepart 2 100%

# 扩展文件系统
resize2fs /dev/sda2

# 验证
df -h /
```

### 2.1 配置国内源

> **Debian 13 (Trixie) 重要变更**：APT 源配置默认使用 deb822 格式（`.sources` 文件），
> 位于 `/etc/apt/sources.list.d/debian.sources`，而非旧的 `/etc/apt/sources.list`。
> 安装器生成的旧 `sources.list` 可能为空或不存在。

```bash
# 先删除/清空旧格式文件（如果存在）
rm -f /etc/apt/sources.list
# 有些安装会留下默认的 debian.sources，先备份再覆盖
cp /etc/apt/sources.list.d/debian.sources /etc/apt/sources.list.d/debian.sources.bak 2>/dev/null

# 使用 deb822 格式配置清华源
cat > /etc/apt/sources.list.d/debian.sources << 'EOF'
Types: deb
URIs: https://mirrors.tuna.tsinghua.edu.cn/debian/
Suites: trixie trixie-updates
Components: main contrib non-free non-free-firmware
Signed-By: /usr/share/keyrings/debian-archive-keyring.gpg

Types: deb
URIs: https://mirrors.tuna.tsinghua.edu.cn/debian-security/
Suites: trixie-security
Components: main contrib non-free non-free-firmware
Signed-By: /usr/share/keyrings/debian-archive-keyring.gpg
EOF

apt update && apt upgrade -y
```

> **说明**：
> - deb822 格式使用 `Key: Value` 多行格式，比旧的单行 `deb ...` 格式更清晰
> - `Signed-By` 字段指定签名密钥，这是 Debian 13 的安全要求
> - 安全源的 URI 是 `debian-security/`，Suite 是 `trixie-security`（不是 `trixie/updates`）
> - 如果你更喜欢旧格式，也可以在 `/etc/apt/sources.list` 中使用传统格式（见下方备选）

<details>
<summary>备选：传统 sources.list 格式（也能用，但非默认）</summary>

```bash
cat > /etc/apt/sources.list << 'EOF'
deb https://mirrors.tuna.tsinghua.edu.cn/debian/ trixie main contrib non-free non-free-firmware
deb https://mirrors.tuna.tsinghua.edu.cn/debian/ trixie-updates main contrib non-free non-free-firmware
deb https://mirrors.tuna.tsinghua.edu.cn/debian-security/ trixie-security main contrib non-free non-free-firmware
EOF
```

注意：使用传统格式时需要先删除 `/etc/apt/sources.list.d/debian.sources`，否则会有重复源。

```bash
# TCP 优化

net.ipv4.tcp_max_syn_backlog = 65536
net.ipv4.tcp_tw_reuse = 1
net.ipv4.tcp_fin_timeout = 30
net.ipv4.tcp_fastopen = 3

# 内存
vm.swappiness = 10
vm.overcommit_memory = 1

# 非本地 IP 绑定（TPROXY 需要）
net.ipv4.ip_nonlocal_bind = 1

# TCP BBR 拥塞控制（代理吞吐量显著提升）
net.core.default_qdisc = fq
net.ipv4.tcp_congestion_control = bbr

# 禁用 IPv6（防止客户端 IPv6 流量绕过 TPROXY）
net.ipv6.conf.all.disable_ipv6 = 1
net.ipv6.conf.default.disable_ipv6 = 1
EOF

sysctl --system
```

### 3.4 nftables TPROXY 规则

先创建公司内网 IP 段配置文件（后续通过 xray-panel 页面维护）：

```bash
cat > /usr/local/xray/office-networks.txt << 'EOF'
10.19.0.0/16
172.16.0.0/16
10.30.0.0/16
10.33.0.0/16
10.16.0.0/16
EOF
```

nftables 使用 set 引用，修改 IP 段只需改一处：

```bash
cat > /usr/local/xray/tproxy.nft << 'EOF'
table ip xray {
    set office_nets {
        type ipv4_addr
        flags interval
        elements = { 10.19.0.0/16, 172.16.0.0/16, 10.30.0.0/16, 10.33.0.0/16, 10.16.0.0/16 }
    }

    chain prerouting {
        type filter hook prerouting priority mangle; policy accept;

        # 公司内网 IP 段优先走 tproxy（引用 set，一处维护）
        ip daddr @office_nets meta l4proto { tcp, udp } tproxy to 127.0.0.1:12345 meta mark set 0x00000001 accept

        # xray 自身流量不代理 (mark 0xff = 255)
        meta mark 0x000000ff return

        # 保留地址段不代理
        ip daddr 127.0.0.0/8 return
        ip daddr 192.168.0.0/16 return
        ip daddr 10.0.0.0/8 return
        ip daddr 172.16.0.0/12 return
        ip daddr 169.254.0.0/16 return
        ip daddr 224.0.0.0/4 return
        ip daddr 240.0.0.0/4 return
        ip daddr 255.255.255.255 return

        # DNS 流量不劫持（MosDNS 作为全局 DNS 直接监听 53 端口）
        meta l4proto { tcp, udp } th dport 53 return

        # 其他所有流量走 xray 代理端口
        meta l4proto { tcp, udp } tproxy to 127.0.0.1:12345 meta mark set 0x00000001 accept
    }

    chain output {
        type route hook output priority mangle; policy accept;

        # 公司内网 IP 段（本机出站也走代理）
        ip daddr @office_nets meta l4proto { tcp, udp } meta mark set 0x00000001 accept

        # xray 自身流量排除
        meta mark 0x000000ff return

        # 保留地址段排除
        ip daddr 127.0.0.0/8 return
        ip daddr 192.168.0.0/16 return
        ip daddr 10.0.0.0/8 return
        ip daddr 172.16.0.0/12 return
        ip daddr 224.0.0.0/4 return

        # DNS 和其他流量
        meta l4proto { tcp, udp } th dport 53 return
        meta l4proto { tcp, udp } meta mark set 0x00000001 accept
    }
}
EOF
```

### 3.5 公司内网 IP 段同步脚本

当通过 xray-panel 修改 `office-networks.txt` 后，调用此脚本热更新 nftables（不重启 xray，不断网）：

```bash
cat > /usr/local/xray/update-office-nets.sh << 'EOF'
#!/bin/bash
# 从 office-networks.txt 热更新 nftables set + 同步 xray config
# 由 xray-panel CGI 调用，或手动执行

NETS_FILE="/usr/local/xray/office-networks.txt"
XRAY_CONF="/usr/local/xray/config.json"

# 读取有效 IP 段（跳过注释和空行）
NETS=$(grep -v '^#' "$NETS_FILE" | grep -v '^$')

if [ -z "$NETS" ]; then
    echo "ERROR: office-networks.txt is empty!"
    exit 1
fi

# 1. 热更新 nftables set（立即生效，不断网）
nft flush set ip xray office_nets 2>/dev/null
for net in $NETS; do
    nft add element ip xray office_nets { $net }
done
echo "[$(date)] nftables office_nets updated: $(echo $NETS | tr '\n' ' ')"

# 2. 同步更新 xray config.json 中的 office IP 列表
# 生成 JSON 格式的 IP 数组
IP_JSON=$(echo "$NETS" | sed 's/.*/"&"/' | paste -sd ',' | sed 's/^/[/;s/$/]/')

# 用 jq 替换 routing rules 中 office 出口的 IP 列表
jq --argjson ips "$IP_JSON" '
  .routing.rules |= map(
    if .outboundTag == "office" and .ip != null then .ip = $ips else . end
  )
' "$XRAY_CONF" > "${XRAY_CONF}.tmp" && mv "${XRAY_CONF}.tmp" "$XRAY_CONF"

echo "[$(date)] xray config.json office IPs updated."
```

### 4.3 MosDNS 配置

```bash
cat > /etc/mosdns/config.yaml << 'EOF'
log:
  level: warn

api:
  http: "127.0.0.1:9091"

plugins:
  # ====== 数据源 ======
  - tag: cn_domains
    type: domain_set
    args:
      files:
        - /etc/mosdns/cn-domains.txt

  - tag: proxy_domains
    type: domain_set
    args:
      files:
        - /etc/mosdns/proxy-domains.txt

  - tag: ad_domains
    type: domain_set
    args:
      files:
        - /etc/mosdns/ad-domains.txt

  - tag: cn_ip
    type: ip_set
    args:
      files:
        - /etc/mosdns/cn-ip.txt

  # ====== 公司内网域名 hosts（返回假 IP 触发 nftables 走 office） ======
  - tag: office_hosts
    type: hosts
    args:
      entries:
        - "domain:hisense.com 10.19.255.253"

  # ====== 上游 DNS ======
  - tag: forward_local
    type: forward
    args:
      concurrent: 2
      upstreams:
        - addr: "223.5.5.5"
        - addr: "119.29.29.29"

  - tag: forward_remote
    type: forward
    args:
      concurrent: 2
      upstreams:
        - addr: "tls://8.8.8.8"
        - addr: "tls://1.1.1.1"

  # ====== 处理序列 ======
  - tag: main_sequence
    type: sequence
    args:
      # 公司内网域名 → 返回假 IP
      - matches: "has_resp"
        exec: accept
      - exec: "$office_hosts"
      - matches: "has_resp"
        exec: accept

      # 广告域名 → 拒绝
      - matches: "qname $ad_domains"
        exec: reject 3

      # 国内域名 → 国内 DNS
      - matches: "qname $cn_domains"
        exec: "$forward_local"
      - matches: "has_resp"
        exec: accept

      # 代理域名 → 境外 DNS
      - matches: "qname $proxy_domains"
        exec: "$forward_remote"
      - matches: "has_resp"
        exec: accept

      # 兜底 → 国内 DNS，如果返回非国内 IP 则用境外 DNS 重查
      - exec: "$forward_local"
      - matches: "!resp_ip $cn_ip"
        exec: "$forward_remote"

  # ====== 入口（全局 DNS，监听所有接口） ======
  - tag: udp_server
    type: udp_server
    args:
      entry: main_sequence
      listen: "0.0.0.0:53"

  - tag: tcp_server
    type: tcp_server
    args:
      entry: main_sequence
      listen: "0.0.0.0:53"
EOF
```

### 4.5 MosDNS 全局 DNS 架构

```
局域网客户端 / 旁路由本机
    │
    ▼ DNS 请求 → 192.168.50.2:53
MosDNS (0.0.0.0:53) — 全局 DNS 入口
    │
    ├── hisense.com → hosts 返回假 IP 10.19.255.253
    │       └→ 客户端连接 10.19.255.253
    │       └→ nftables 匹配 office_nets → tproxy → Xray
    │       └→ sniffing 还原域名 → office 出口 → 公司内网
    │
    ├── 广告域名 → 拒绝 (NXDOMAIN)
    ├── 国内域名 → 223.5.5.5 / 119.29.29.29
    └── 国外域名 → DoT 8.8.8.8 / 1.1.1.1
```

> **设计要点**：
> - MosDNS 监听 0.0.0.0:53，统一处理全屋 + 本机的 DNS 请求
> - nftables 中 DNS 端口 (53) 标记为 return（不劫持），让请求直达 MosDNS
> - Xray 不再处理 DNS，专注做流量代理
> - 公司域名通过 MosDNS hosts 返回假 IP，触发 nftables office_nets 规则进入 Xray

将旁路由本机 DNS 指向 MosDNS：

```bash
# 修改 /etc/resolv.conf
cat > /etc/resolv.conf << 'EOF'
nameserver 127.0.0.1
EOF

# 防止被 DHCP 覆盖
chattr +i /etc/resolv.conf
```

## 第九部分：安全加固

### 9.1 防火墙（保护旁路由自身）

```bash
cat > /etc/nftables.conf << 'EOF'
#!/usr/sbin/nft -f
# 只清理自己的 table，不影响 xray 的 tproxy 规则
table inet filter
delete table inet filter

table inet filter {
    chain input {
        type filter hook input priority filter; policy drop;

        # 已建立连接
        ct state established,related accept

        # 本地回环
        iif lo accept

        # ICMP
        ip protocol icmp accept

        # SSH (仅局域网)
        ip saddr 192.168.50.0/24 tcp dport 22 accept

        # DNS (MosDNS)
        ip saddr 192.168.50.0/24 tcp dport 53 accept
        ip saddr 192.168.50.0/24 udp dport 53 accept

        # Xray TPROXY 端口（来自局域网转发）
        ip saddr 192.168.50.0/24 tcp dport 12345 accept
        ip saddr 192.168.50.0/24 udp dport 12345 accept

        # Nginx (Xray 管理面板)
        ip saddr 192.168.50.0/24 tcp dport 80 accept

        # Xray HTTP/SOCKS/VLESS 代理入口
        ip saddr 192.168.50.0/24 tcp dport { 1080, 10800, 16825 } accept

        # 记录丢弃的包（调试用，正式环境可注释掉）
        # log prefix "INPUT DROP: " limit rate 5/minute
    }

    chain forward {
        type filter hook forward priority filter; policy accept;
    }

    chain output {
        type filter hook output priority filter; policy accept;
    }
}
EOF

# 注意：这个防火墙规则和 xray 的 tproxy.nft 是两个独立的 table，互不冲突
# 使用 "delete table inet filter" 而非 "flush ruleset"，确保不会误删 xray 的 table ip xray
systemctl enable nftables
```

### 10.3 扩展分区到整块 SSD

由于虚拟磁盘只有 50GB，写入 1TB SSD 后有大量未分配空间，需要扩展。
倍控的 SSD 是 NVMe 设备（`/dev/nvme0n1`），且有 swap 分区挡在根分区后面，需要先删 swap：

```bash

# 删除并重建根分区（p2），从原始起始扇区到磁盘末尾
# 注意：1980416 是 p2 的原始起始扇区，0 表示到磁盘末尾
sgdisk -d 2 -n 2:1980416:0 -t 2:8300 /dev/nvme0n1

partprobe /dev/nvme0n1

# 在线扩展文件系统

# 验证（应该显示接近 938GB）
```

```bash
ssh root@192.168.50.2
```

> 确认能登录后再继续！不通就停下排查，不要往下走。

**第三步：持久化静态 IP 配置**

```bash
cat > /etc/network/interfaces << 'EOF'
auto lo
iface lo inet loopback

auto enp3s0
iface enp3s0 inet static
    address 192.168.50.2
    netmask 255.255.255.0
    gateway 192.168.50.1
    dns-nameservers 127.0.0.1
EOF
```

**第四步：禁用 systemd-networkd 并重启**

```bash
systemctl disable systemd-networkd systemd-networkd-wait-online
reboot
```

**第五步：重启后验证**

```bash
ssh root@192.168.50.2
# 确认只有静态 IP，没有 DHCP 地址
ip addr show enp3s0 | grep inet
# 应该只有 192.168.50.2/24
```

### 10.6 切换全屋网关

确认所有测试通过后，到华硕 BE86U 管理界面：

1. 进入 **LAN** → **DHCP Server**
2. **Default Gateway** 设置为 `192.168.50.2`
3. **DNS Server** 设置为 `192.168.50.2`
4. 保存应用

全屋设备续约 DHCP 后，流量自动走旁路由。

## 注意事项

1. **4G 内存限制**: 跑 Xray + MosDNS + Nginx 绰绰有余，日常内存占用不到 500MB
2. **日志管理**: access.log 为 info 级别，logrotate 每天轮转保留 7 天，不会撑爆磁盘
3. **散热**: Celeron 7305 无风扇被动散热，夏天注意摆放位置通风
4. **备份配置**: 建议把 `/usr/local/xray/` 和 `/etc/mosdns/` 定期备份到群晖 NAS
