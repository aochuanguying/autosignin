# 倍控 G31 N7305 旁路由完整部署方案

## 概述

- **设备**: 倍控 G31，Intel Celeron 7305 (5核6线程, 1.00GHz)，4×2.5G i226-V 网口 + 1×RJ45 COM 串口，1T SSD，4G DDR4
- **目标系统**: Debian 13 (Trixie) minimal
- **角色**: 专职旁路由 — 透明代理 + DNS 分流 + 去广告
- **新 IP**: 192.168.50.2（接替原 iStoreOS 旁路由）
- **网段**: 192.168.50.0/24，主路由 192.168.50.1（华硕 BE86U）

## 网络拓扑

```
光猫(桥接) → BE86U WAN(2.5G) 
                │
                ▼
        BE86U LAN(2.5G) → 兮克交换机(2.5G)
                                │
                ├── 倍控 G31 eth0 (192.168.50.2) ← 旁路由
                ├── 群晖 218+ (2.5G口)           ← NAS
                ├── 联想 X5-14 (1G口)            ← 服务器
                └── 其他有线设备
```

## 第一部分：通过 VirtualBox 直通 SSD 安装 Debian 13

利用联想 X5-14（Win11）的 VirtualBox 虚拟机，将 Debian 直接安装到倍控的 SSD 上，无需键盘和显示器连接倍控。

### 1.1 在联想上下载所需软件

**VirtualBox**：https://www.virtualbox.org/wiki/Downloads → 下载 "Windows hosts" 版本，安装。

**Debian 13 ISO**：https://mirrors.tuna.tsinghua.edu.cn/debian-cd/current/amd64/iso-cd/
下载 `debian-13.6.0-amd64-netinst.iso`（755MB，选第一个，不要 edu 或 mac 版本）。

### 1.2 确认 SSD 磁盘号

1. SSD 硬盘盒通过 USB 连接联想电脑
2. 右键「开始菜单」→「磁盘管理」
3. 找到 1TB 的 SSD，记下磁盘号（如「磁盘 1」）

> ⚠️ **确认磁盘号正确！选错会覆盖联想系统盘。**

### 1.3 创建 VirtualBox 原始磁盘映射

以**管理员身份**打开命令提示符（CMD）：

```cmd
cd "C:\Program Files\Oracle\VirtualBox"

:: 假设 SSD 是磁盘1，改成你实际的编号
VBoxManage internalcommands createrawvmdk -filename "C:\Users\你的用户名\ssd.vmdk" -rawdisk \\.\PhysicalDrive1
```

成功提示：`RAW host disk access VMDK file ... created successfully.`

### 1.4 创建虚拟机

**以管理员身份运行 VirtualBox**，然后：

1. 点击「新建」：
   - 名称：`gateway`
   - 类型：Linux → Debian (64-bit)
   - 内存：2048 MB
   - 硬盘：「使用已有虚拟硬盘文件」→ 选择 `ssd.vmdk`

2. 进入虚拟机「设置」：
   - **系统** → 启动顺序：光驱第一
   - **系统** → 勾选「启用 EFI」
   - **存储** → 光驱 → 选择 Debian ISO 文件
   - **网络** → 连接方式：桥接网卡

### 1.5 安装 Debian 13

启动虚拟机，选择 "Graphical Install"：

| 步骤 | 选择 |
|------|------|
| Language | English |
| Location | China |
| Keyboard | American English |
| Hostname | `gateway` |
| Domain | 留空 |
| Root password | 设置强密码 |
| User | 不创建普通用户（只用 root） |
| Partitioning | **Guided - use entire disk** |
| Partition scheme | All files in one partition |
| Mirror | `mirrors.tuna.tsinghua.edu.cn` |
| Software selection | **只勾选 SSH server + standard system utilities** |
| GRUB | Yes，安装到 SSD 上 |

### 1.6 配置兜底网络（关键！）

安装完成后虚拟机会重启进入 Debian，用 root 登录，执行：

```bash
# 禁用 ifupdown 的固定网口配置
cat > /etc/network/interfaces << 'EOF'
auto lo
iface lo inet loopback
EOF

# 用 systemd-networkd 匹配所有以太网口自动 DHCP
cat > /etc/systemd/network/20-wired.network << 'EOF'
[Match]
Type=ether

[Network]
DHCP=yes
EOF

# 启用 systemd-networkd
systemctl enable systemd-networkd

# 确认 SSH 开机启动 + 允许 root 登录
sed -i 's/#PermitRootLogin prohibit-password/PermitRootLogin yes/' /etc/ssh/sshd_config
systemctl enable ssh
```

> 不要关机！接下来在虚拟机中完成所有配置。

### 3.4 nftables TPROXY 规则

先创建公司内网 IP 段配置文件（后续通过 xray-panel 页面维护）：

```bash
cat > /usr/local/xray/office-networks.txt << 'EOF'
10.19.0.0/16
172.16.0.0/16
10.30.0.0/16
10.33.0.0/16
10.16.0.0/16
EOF
```

nftables 使用 set 引用，修改 IP 段只需改一处：

```bash
cat > /usr/local/xray/tproxy.nft << 'EOF'
table ip xray {
    set office_nets {
        type ipv4_addr
        flags interval
        elements = { 10.19.0.0/16, 172.16.0.0/16, 10.30.0.0/16, 10.33.0.0/16, 10.16.0.0/16 }
    }

    chain prerouting {
        type filter hook prerouting priority mangle; policy accept;

        # 公司内网 IP 段优先走 tproxy（引用 set，一处维护）
        ip daddr @office_nets meta l4proto { tcp, udp } tproxy to 127.0.0.1:12345 meta mark set 0x00000001 accept

        # xray 自身流量不代理 (mark 0xff = 255)
        meta mark 0x000000ff return

        # 保留地址段不代理
        ip daddr 127.0.0.0/8 return
        ip daddr 192.168.0.0/16 return
        ip daddr 10.0.0.0/8 return
        ip daddr 172.16.0.0/12 return
        ip daddr 169.254.0.0/16 return
        ip daddr 224.0.0.0/4 return
        ip daddr 240.0.0.0/4 return
        ip daddr 255.255.255.255 return

        # DNS 流量不劫持（MosDNS 作为全局 DNS 直接监听 53 端口）
        meta l4proto { tcp, udp } th dport 53 return

        # 其他所有流量走 xray 代理端口
        meta l4proto { tcp, udp } tproxy to 127.0.0.1:12345 meta mark set 0x00000001 accept
    }

    chain output {
        type route hook output priority mangle; policy accept;

        # 公司内网 IP 段（本机出站也走代理）
        ip daddr @office_nets meta l4proto { tcp, udp } meta mark set 0x00000001 accept

        # xray 自身流量排除
        meta mark 0x000000ff return

        # 保留地址段排除
        ip daddr 127.0.0.0/8 return
        ip daddr 192.168.0.0/16 return
        ip daddr 10.0.0.0/8 return
        ip daddr 172.16.0.0/12 return
        ip daddr 224.0.0.0/4 return

        # DNS 和其他流量
        meta l4proto { tcp, udp } th dport 53 return
        meta l4proto { tcp, udp } meta mark set 0x00000001 accept
    }
}
EOF
```

### 3.5 公司内网 IP 段同步脚本

当通过 xray-panel 修改 `office-networks.txt` 后，调用此脚本热更新 nftables（不重启 xray，不断网）：

```bash
cat > /usr/local/xray/update-office-nets.sh << 'EOF'
#!/bin/bash
# 从 office-networks.txt 热更新 nftables set + 同步 xray config
# 由 xray-panel CGI 调用，或手动执行

NETS_FILE="/usr/local/xray/office-networks.txt"
XRAY_CONF="/usr/local/xray/config.json"

# 读取有效 IP 段（跳过注释和空行）
NETS=$(grep -v '^#' "$NETS_FILE" | grep -v '^$')

if [ -z "$NETS" ]; then
    echo "ERROR: office-networks.txt is empty!"
    exit 1
fi

# 1. 热更新 nftables set（立即生效，不断网）
nft flush set ip xray office_nets 2>/dev/null
for net in $NETS; do
    nft add element ip xray office_nets { $net }
done
echo "[$(date)] nftables office_nets updated: $(echo $NETS | tr '\n' ' ')"

# 2. 同步更新 xray config.json 中的 office IP 列表
# 生成 JSON 格式的 IP 数组
IP_JSON=$(echo "$NETS" | sed 's/.*/"&"/' | paste -sd ',' | sed 's/^/[/;s/$/]/')

# 用 jq 替换 routing rules 中 office 出口的 IP 列表
jq --argjson ips "$IP_JSON" '
  .routing.rules |= map(
    if .outboundTag == "office" and .ip != null then .ip = $ips else . end
  )
' "$XRAY_CONF" > "${XRAY_CONF}.tmp" && mv "${XRAY_CONF}.tmp" "$XRAY_CONF"

echo "[$(date)] xray config.json office IPs updated."
echo "[$(date)] Run 'systemctl restart xray' to apply xray config changes."
EOF

chmod +x /usr/local/xray/update-office-nets.sh
```

> **使用方式**：在 xray-panel 页面编辑 office-networks.txt 保存后，面板调用此脚本。
> nftables 立即生效（热更新），xray config 需要重启才生效（面板点击重启按钮）。

### 3.6 Xray systemd 服务

```bash
cat > /etc/systemd/system/xray.service << 'EOF'
[Unit]
Description=Xray TPROXY Transparent Proxy
After=network.target nftables.service
Wants=network.target

[Service]
Type=simple
ExecStartPre=/sbin/ip route add local default dev lo table 100 || true
ExecStartPre=/sbin/ip rule add fwmark 1 table 100 || true
ExecStartPre=/usr/sbin/nft -f /usr/local/xray/tproxy.nft
ExecStart=/usr/local/xray/xray run -c /usr/local/xray/config.json
ExecStopPost=/usr/sbin/nft delete table ip xray || true
ExecStopPost=/sbin/ip rule del fwmark 1 table 100 || true
ExecStopPost=/sbin/ip route del local default dev lo table 100 || true
Restart=on-failure
RestartSec=5
LimitNOFILE=1048576

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable xray
systemctl start xray
systemctl status xray
```

### 3.7 Xray 日志目录 + 自动轮转

```bash

    notifempty
```

> `copytruncate` 不需要重启 xray 就能轮转日志。

### 3.8 GeoData 自动更新脚本

```bash
cat > /usr/local/xray/update-geodata.sh << 'EOF'
#!/bin/bash
# 更新 GeoIP 和 GeoSite 数据库

cd /usr/local/xray

echo "[$(date)] Updating GeoIP..."
wget -q --timeout=60 -O geoip.dat.new https://github.com/Loyalsoldier/v2ray-rules-dat/releases/latest/download/geoip.dat
if [ $? -eq 0 ] && [ -s geoip.dat.new ]; then
    mv geoip.dat.new geoip.dat
    echo "[$(date)] GeoIP updated."
else
    rm -f geoip.dat.new
    echo "[$(date)] GeoIP update failed."
fi

echo "[$(date)] Updating GeoSite..."
wget -q --timeout=60 -O geosite.dat.new https://github.com/Loyalsoldier/v2ray-rules-dat/releases/latest/download/geosite.dat
if [ $? -eq 0 ] && [ -s geosite.dat.new ]; then
    mv geosite.dat.new geosite.dat
    echo "[$(date)] GeoSite updated."
else
    rm -f geosite.dat.new
    echo "[$(date)] GeoSite update failed."
fi

# 重启 xray 加载新数据
systemctl restart xray
echo "[$(date)] Xray restarted."
EOF

chmod +x /usr/local/xray/update-geodata.sh
```


### 4.3 MosDNS 配置

```bash
cat > /etc/mosdns/config.yaml << 'EOF'
log:
  level: warn

api:
  http: "127.0.0.1:9091"

plugins:
  # ====== 数据源 ======
  - tag: cn_domains
    type: domain_set
    args:
      files:
        - /etc/mosdns/cn-domains.txt

  - tag: proxy_domains
    type: domain_set
    args:
      files:
        - /etc/mosdns/proxy-domains.txt

  - tag: ad_domains
    type: domain_set
    args:
      files:
        - /etc/mosdns/ad-domains.txt

  - tag: cn_ip
    type: ip_set
    args:
      files:
        - /etc/mosdns/cn-ip.txt

  # ====== 公司内网域名 hosts（返回假 IP 触发 nftables 走 office） ======
  - tag: office_hosts
    type: hosts
    args:
      entries:
        - "domain:hisense.com 10.19.255.253"

  # ====== 上游 DNS ======
  - tag: forward_local
    type: forward
    args:
      concurrent: 2
      upstreams:
        - addr: "223.5.5.5"
        - addr: "119.29.29.29"

  - tag: forward_remote
    type: forward
    args:
      concurrent: 2
      upstreams:
        - addr: "tls://8.8.8.8"
        - addr: "tls://1.1.1.1"

  # ====== 处理序列 ======
  - tag: main_sequence
    type: sequence
    args:
      # 公司内网域名 → 返回假 IP
      - matches: "has_resp"
        exec: accept
      - exec: "$office_hosts"
      - matches: "has_resp"
        exec: accept

      # 广告域名 → 拒绝
      - matches: "qname $ad_domains"
        exec: reject 3

      # 国内域名 → 国内 DNS
      - matches: "qname $cn_domains"
        exec: "$forward_local"
      - matches: "has_resp"
        exec: accept

      # 代理域名 → 境外 DNS
      - matches: "qname $proxy_domains"
        exec: "$forward_remote"
      - matches: "has_resp"
        exec: accept

      # 兜底 → 国内 DNS，如果返回非国内 IP 则用境外 DNS 重查
      - exec: "$forward_local"
      - matches: "!resp_ip $cn_ip"
        exec: "$forward_remote"

  # ====== 入口（全局 DNS，监听所有接口） ======
  - tag: udp_server
    type: udp_server
    args:
      entry: main_sequence
      listen: "0.0.0.0:53"

  - tag: tcp_server
    type: tcp_server
    args:
      entry: main_sequence
      listen: "0.0.0.0:53"
EOF
```

### 4.4 MosDNS systemd 服务

```bash
cat > /etc/systemd/system/mosdns.service << 'EOF'
[Unit]
Description=MosDNS DNS Server
After=network.target
Before=xray.service

[Service]
Type=simple
ExecStart=/usr/local/bin/mosdns start -c /etc/mosdns/config.yaml
Restart=on-failure
RestartSec=5

[Install]

EOF

# 先停掉系统自带的 systemd-resolved（如果有）

systemctl daemon-reload

```

### 4.5 MosDNS 全局 DNS 架构

```
局域网客户端 / 旁路由本机
    │
    ▼ DNS 请求 → 192.168.50.2:53
MosDNS (0.0.0.0:53) — 全局 DNS 入口
    │
    ├── hisense.com → hosts 返回假 IP 10.19.255.253
    │       └→ 客户端连接 10.19.255.253
    │       └→ nftables 匹配 office_nets → tproxy → Xray
    │       └→ sniffing 还原域名 → office 出口 → 公司内网
    │
    ├── 广告域名 → 拒绝 (NXDOMAIN)
    ├── 国内域名 → 223.5.5.5 / 119.29.29.29
    └── 国外域名 → DoT 8.8.8.8 / 1.1.1.1
```

> **设计要点**：
> - MosDNS 监听 0.0.0.0:53，统一处理全屋 + 本机的 DNS 请求
> - nftables 中 DNS 端口 (53) 标记为 return（不劫持），让请求直达 MosDNS
> - Xray 不再处理 DNS，专注做流量代理
> - 公司域名通过 MosDNS hosts 返回假 IP，触发 nftables office_nets 规则进入 Xray

将旁路由本机 DNS 指向 MosDNS：

```bash
# 修改 /etc/resolv.conf
cat > /etc/resolv.conf << 'EOF'
nameserver 127.0.0.1
EOF

# 防止被 DHCP 覆盖
chattr +i /etc/resolv.conf
```

## 第九部分：安全加固

### 9.1 防火墙（保护旁路由自身）

```bash
cat > /etc/nftables.conf << 'EOF'
#!/usr/sbin/nft -f
# 只清理自己的 table，不影响 xray 的 tproxy 规则
table inet filter
delete table inet filter

table inet filter {
    chain input {
        type filter hook input priority filter; policy drop;

        # 已建立连接
        ct state established,related accept

        # 本地回环
        iif lo accept

        # ICMP
        ip protocol icmp accept

        # SSH (仅局域网)
        ip saddr 192.168.50.0/24 tcp dport 22 accept

        # DNS (MosDNS)
        ip saddr 192.168.50.0/24 tcp dport 53 accept
        ip saddr 192.168.50.0/24 udp dport 53 accept

        # Xray TPROXY 端口（来自局域网转发）
        ip saddr 192.168.50.0/24 tcp dport 12345 accept
        ip saddr 192.168.50.0/24 udp dport 12345 accept

        # Nginx (Xray 管理面板)
        ip saddr 192.168.50.0/24 tcp dport 80 accept

        # Xray HTTP/SOCKS/VLESS 代理入口
        ip saddr 192.168.50.0/24 tcp dport { 1080, 10800, 16825 } accept

        # 记录丢弃的包（调试用，正式环境可注释掉）
        # log prefix "INPUT DROP: " limit rate 5/minute
    }

    chain forward {
        type filter hook forward priority filter; policy accept;
    }

    chain output {
        type filter hook output priority filter; policy accept;
    }
}
EOF

# 注意：这个防火墙规则和 xray 的 tproxy.nft 是两个独立的 table，互不冲突
# 使用 "delete table inet filter" 而非 "flush ruleset"，确保不会误删 xray 的 table ip xray
systemctl enable nftables
```
